# Оценка мощностей компьютеров Салона 14

## Методология

### Факторы оценки

Оценка основана на трёх ключевых компонентах, доступных из DxDiag-отчётов:

| Компонент | Вес | Обоснование |
|---|---|---|
| **Процессор** | 40% | Определяет общую вычислительную способность: количество ядер, потоков, архитектура (IPC), тактовая частота |
| **ОЗУ** | 30% | Ограничивает количество одновременно запущенных приложений и объём обрабатываемых данных |
| **Видеокарта** | 20% | Определяет графическую производительность, аппаратное декодирование видео, поддержку современных API |
| **DirectX** | 10% | Индикатор совместимости с современным ПО и играми (DX12 vs DX11) |

### Ограничения методологии

- **Нет данных о накопителях**: скорость диска (HDD vs SSD) критична для отзывчивости системы, но отсутствует в DxDiag
- **Нет данных о температуре и состоянии**: перегрев может снижать производительность на 20–40%
- **Нет бенчмарков**: оценки основаны на архитектурных характеристиках, а не на реальных измерениях
- **Нет данных о сети**: для клиентских задач (CRM, 1C) скорость сети важна
- **Производительность IPC**: архитектура Core2 Duo (2008) значительно уступает Ivy Bridge (2012) при одинаковой частоте

## Итоговые группы

### Группа A — Мощные (серверные платформы)

**a1, a2, a5, k1, k2** — Xeon E5, 16+ ГБ ОЗУ. Подходят для любых задач, включая тяжёлые вычисления и видеомонтаж.

### Группа B — Рабочие (настольные i5/i3 + 12–32 ГБ)

**a3, a4, a6, a7, a8, a9, a11, c6, c7, c9** — Достаточны для офиса, CRM, базового дизайна. Требуют миграции с Win7.

### Группа C — Базовые (Pentium/i3 + 4–10 ГБ)

**a10, c2, c3, c5, c8, c10** — Только для простейших задач (офис, почта, браузер). Ограничены 4 ГБ ОЗУ.

### Группа D — Устаревшие (замена)

**c4** — Не соответствует минимальным требованиям современного ПО.
